🚀 Como publicar o Painel OKR no Vercel

Guia passo a passo — sem instalar nada no computador  ·  ~10 minutos

📦
O que você tem neste ZIP:
index.html — o painel completo   vercel.json — configuração do Vercel   GUIA.html — este guia
PASSO 1

Crie uma conta no Vercel

Acesse vercel.com e clique em Sign Up. Use sua conta Google — é mais rápido.

🌐 vercel.com → Sign Up → Continue with Google
✅ O plano gratuito do Vercel suporta projetos ilimitados e não tem limite de acessos para projetos internos.
PASSO 2

Crie um novo projeto e faça upload dos arquivos

Dentro do Vercel, clique em Add New → Project. Na tela seguinte, procure o botão "Browse" ou "Upload" e selecione a pasta descompactada deste ZIP (não o ZIP em si).

Vercel Dashboard → Add New → Project → Upload folder → selecione a pasta "vercel-deploy"
  • O Vercel vai detectar automaticamente o index.html como página principal
  • O vercel.json já está configurado para roteamento correto
  • Clique em Deploy — leva menos de 1 minuto
PASSO 3

Copie a URL gerada pelo Vercel

Após o deploy, o Vercel mostra uma URL como:

https://painel-okr-cpp-abc123.vercel.app

Você pode trocar este nome depois em Project Settings → Domains. Copie essa URL — ela será usada no próximo passo.

⚠️
Esta URL precisa ser cadastrada no Firebase para que o login com Google funcione. Sem isso, o painel abre mas o botão de login retorna erro.
PASSO 4 — CRÍTICO

Autorize a URL no Firebase Authentication

Este passo é obrigatório. Sem ele, o login com Google será bloqueado.

  • Acesse console.firebase.google.com
  • Selecione o projeto planejamento-integracao
  • No menu lateral: Authentication → Settings → Authorized domains
  • Clique em Add domain
  • Cole a URL do Vercel sem o https://, por exemplo:
    painel-okr-cpp-abc123.vercel.app
  • Clique em Add
Firebase Console → Authentication → Settings → Authorized domains → Add domain
✅ Se você escolher um domínio personalizado no Vercel (ex: okr.pequenopríncipe.org.br), adicione esse domínio também aqui.
PASSO 5

Crie o banco Firestore (se ainda não fez)

Se ainda não criou o banco de dados:

  • Firebase Console → Firestore Database → Criar banco de dados
  • Modo: Produção
  • Região: southamerica-east1 (São Paulo — menor latência para o CPP)
✅ Se o banco já existe, pode pular este passo.
PASSO 6

Publique as regras de segurança do Firestore

As regras garantem que só usuários autenticados acessem os dados.

  • Firebase Console → Firestore Database → Regras
  • Substitua o conteúdo pelas regras abaixo e clique em Publicar
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
💡 Esta é a regra básica — qualquer usuário autenticado pode ler e escrever. Para restrições por domínio de e-mail (ex: só @pequenopríncipe.org.br), entre em contato para gerar uma regra mais específica.
PASSO 7

Teste o painel e compartilhe

Abra a URL do Vercel no navegador. Você deve ver a tela de login. Clique em Entrar com Google e pronto — o painel estará funcionando e sincronizando com o Firestore em tempo real.

🎉 Para compartilhar com a equipe, basta enviar a URL. Qualquer pessoa com uma conta Google poderá fazer login e colaborar no painel.
CHECKLIST FINAL

Antes de compartilhar, confirme:

  • ✓
    Conta Vercel criada e projeto publicado
  • ✓
    URL do Vercel copiada
  • ✓
    URL cadastrada em Firebase → Authentication → Authorized domains
  • ✓
    Banco Firestore criado (modo produção, região São Paulo)
  • ✓
    Regras de segurança publicadas no Firestore
  • ✓
    Login com Google testado e funcionando
  • ✓
    URL compartilhada com a equipe da Comissão de Integração
PROBLEMAS COMUNS

Se algo não funcionar

  • Erro "auth/unauthorized-domain" → O domínio do Vercel não foi adicionado no Firebase Auth (Passo 4)
  • Tela em branco após login → Abra o console do navegador (F12) e veja o erro; geralmente é regra do Firestore
  • Dados não aparecem → Verifique se as regras do Firestore foram publicadas (Passo 6)
  • Login não abre popup → Verifique se o Google foi habilitado em Firebase → Authentication → Sign-in method
  • URL personalizada não funciona → Adicione o domínio personalizado também no Firebase Auth
ATUALIZAÇÕES FUTURAS

Como atualizar o painel depois

Sempre que receber uma nova versão do arquivo index.html:

  • Acesse vercel.com → seu projeto → Deployments
  • Clique em Deploy → selecione apenas o novo index.html
  • O Vercel faz o deploy em segundos — a URL permanece a mesma
💡 Os dados no Firestore não são afetados por atualizações do painel. Atualizar o HTML é seguro.
